# <span data-lang="en">Workshop 2 - DEMO Mermaid Documentation</span><span data-lang="fr">Atelier 2 - DEMO Documentation Mermaid</span>

## <span data-lang="en">Description</span><span data-lang="fr">Description</span>
**<span data-lang="en">Objective</span><span data-lang="fr">Objectif</span>:** <span data-lang="en">Show how AI can automatically generate diagrams and flows.</span><span data-lang="fr">Montrer comment l'IA peut générer automatiquement des diagrammes et des flux.</span>

### <span data-lang="en">Example prompt:</span><span data-lang="fr">Exemple de prompt :</span>
```
You are a software architect.
Create documentation for a simple employee and work schedule management web application including:
- Navigation flow diagram (Mermaid)
- Sequence diagram for the work schedule process
- Main use cases
- Main use cases diagram
- Markdown structure organized by sections
Explain each diagram and how it relates to the use cases.
```

### <span data-lang="en">Suggested supplementary prompt:</span><span data-lang="fr">Prompt complémentaire suggéré :</span>
```
You are a technical writer and software architect.
Take the previous documentation, create a version 2.0 and improve it by:
- Adding detailed descriptions for each use case, including actors, preconditions, and postconditions.
- Creating an Entity-Relationship Diagram (ERD) in Mermaid for the database structure.
- Adding a state diagram for the employee work schedule lifecycle.
- Including a section with best practices for scalability and security in the application.
- Organizing everything in a clear Markdown structure with headings, subheadings, and code blocks for diagrams.
Explain how each new diagram complements the previous ones and why these details are important for developers and stakeholders.
```
